New Ticket

This window is where you create a new ticket or issue.

Ticket

The following are some of the key fields available for a new ticket:



Name: Each ticket must have a name/description. This is typically the subject of the ticket.



Ticket Type: Tickets can be classified as any Ticket Type you have defined and can be reclassified at anytime.



Status: The Status defines the current state of the ticket, and the various status can be customized in the Admin section. Additionally, workflows can be defined so that tickets move betweens statuses in an orderly fashion.



Severity: The Severity of a ticket helps define what kind of attention is needed. Of course, these may also be customized within the Admin section.



Product (Enterprise Only): By defining the products provided by your company within the Products section, you will have the option to associate the ticket with a specific product.



Assigned To: Assigns the ticket to a defined TeamSupport user.



Assigned Group: Like user assignment, you also have the option to assign a ticket to groups.



Customer: This is where you associate the ticket with one or more customers.



Visible to Customers: If this option is selected, any customers assocaited with the ticket will be notified via e-mail that the ticket was created, and of any subsequent changes or actions that are added to the ticket.
